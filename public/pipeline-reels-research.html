Инструкция для маркетинг-агента

Пайплайн: залетевшие рилсы → сценарий в твоём стиле

Как агент сам находит вирусные рилсы конкурентов, расшифровывает их и переписывает под твой Tone of Voice. Настраиваешь один раз — дальше присылаешь список конкурентов и получаешь готовые сценарии.

Что на выходе По каждому залетевшему рилсу — дословный транскрипт + сценарий-телесуфлёр в твоём Tone of Voice (что говорить на камеру) + ТЗ монтажёру (раскадровка, плашки, тайминги). Тебе остаётся снять и смонтировать.

Как это работает — в двух словах

Агент берёт аккаунты конкурентов → через сервис HikerAPI вытягивает их рилсы и сортирует по просмотрам → скачивает самые залетевшие → извлекает звук → расшифровывает в текст → переводит (если на другом языке) → переписывает под твой стиль → отдаёт сценарий + ТЗ монтажёру. Всё автоматически, руками ничего скачивать не надо.

Это делается один раз. Дальше просто пользуешься.

Часть 2. Что даёшь агенту каждый раз

  • Список конкурентов — 3–7 аккаунтов Instagram в твоей нише (чьи рилсы хочешь разобрать). Просто ники, например @nick1, @nick2.
  • Сколько рилсов взять и за какой период (например «топ-5 за последний месяц по просмотрам»).
  • (один раз) Твой Tone of Voice — если ещё не давала.

Всё. Дальше агент делает сам.

Часть 3. Пайплайн по шагам (что делает агент)

  1. Находит аккаунты конкурентов
    По нику получает ID аккаунта через HikerAPI. Если нужны новые конкуренты — ищет похожие по хэштегам ниши.
  2. Собирает их рилсы и метрики
    Тянет последние публикации каждого аккаунта с числом просмотров, лайков, комментариев.
  3. Отбирает залетевшие
    Сортирует по просмотрам, берёт топ (например топ-5). Именно вирусные — то, что уже проверено аудиторией, а не случайные ролики.
  4. Скачивает видео
    Забирает файлы отобранных рилсов по прямым ссылкам.
  5. Достаёт звук
    Через ffmpeg извлекает аудиодорожку из видео (в mp3/wav).
  6. Расшифровывает в текст
    Отправляет звук в Groq (Whisper) → получает дословный транскрипт того, что говорят в ролике.
  7. Переводит (если надо)
    Если рилс на английском/другом языке — переводит на русский, сохраняя смысл и структуру.
  8. Переписывает в твой Tone of Voice
    Не тупой перевод, а адаптация: тот же цепляющий скелет (хук → польза → призыв), но твоим голосом, твоими словами, под твою нишу. Убирает чужой пафос, добавляет твои сигнатурные фразы.
  9. Отдаёт готовое
    По каждому рилсу: ссылка + метрики + транскрипт + сценарий-телесуфлёр (что говорить) + ТЗ монтажёру (раскадровка, плашки, тайминги).
Почему это работает Ты не гадаешь, что снять. Берёшь то, что УЖЕ залетело у конкурентов (проверено просмотрами), и переупаковываешь под себя. Риск «снял, а не зашло» падает в разы.

Выдай по каждому рилсу: — ссылку на оригинал + метрики (просмотры) — дословный транскрипт — сценарий-телесуфлёр (что говорить на камеру, по моему TOV) — ТЗ монтажёру (раскадровка по секундам, плашки, тайминги, формат 9:16) Оформи всё одним аккуратным документом.

Часть 5. Технические детали (для тех-агента)

Если агент технический — вот опорные точки, остальное он допишет сам:

  • HikerAPI: получить user id по нику → вытянуть медиа аккаунта → в ответе поле числа просмотров (play_count) → сортировать по нему. Есть поиск аккаунтов и рилсов по хэштегам ниши для расширения списка конкурентов.
  • Скачивание: прямая ссылка на видео из ответа HikerAPI → сохранить файл.
  • Аудио: ffmpeg -i reel.mp4 -vn -acodec libmp3lame reel.mp3
  • Транскрипт: Groq API, модель whisper-large-v3, отправить аудиофайл → получить текст.
  • Автоматизация: можно поставить на расписание (раз в неделю) — агент сам собирает свежие залетевшие и присылает пачку сценариев.
Пайплайн-инструкция для учеников интенсива «AI-команда на Claude». Ключи HikerAPI/Groq — личные у каждого, хранятся защищённо. Сервисы: hikerapi.com (сбор рилсов), console.groq.com (расшифровка). Сценарии всегда проходят через личный Tone of Voice — это ядро, без него будет чужой голос.
