Словарь терминов

Все слова из уроков простыми словами: что значит и зачем. Держи под рукой — с ним уроки перестают казаться набором непонятных команд.

Главное

AI-агентоснова
Программа на основе AI, которая не просто отвечает, а САМ действует: пользуется инструментами, проверяет себя, помнит прошлое, доводит задачу до результата.
Зачем: чтобы переложить на него рутину — он работает, ты занимаешься главным.
Аналогия: не «собеседник», а «сотрудник».
ClaudeAI
Искусственный интеллект от компании Anthropic — «мозг», на котором мы строим агентов. Один из самых сильных в мире на 2026 год.
Зачем: это интеллект, который думает за агента. Берём его как основу.
Claude Codeпрограмма
Программа от Anthropic, через которую агент «живёт» на компьютере. Работает в терминале. На ней мы строим агентов.
Зачем: это «тело» агента — то, что выполняет команды и держит его запущенным.
Модель (Opus / Sonnet / Haiku)мозг
«Мозг» агента — то, что думает. Внутри Claude их три по силе: Opus (сильная, для сложного), Sonnet (средняя, повседневная), Haiku (простая, мелочи).
Зачем: под задачу берёшь нужную силу. Начинаем с сильной (Opus).

Установка и работа

Терминалинструмент
Чёрное окно, куда команды вводят текстом, а не кнопками. Бывает отдельной программой («Терминал» на Mac) или встроенным в VS Code — это одно и то же.
Зачем: через него ставят и запускают программы. Мы просто вставляем готовые команды и жмём Enter.
VS Codeредактор
Бесплатная программа-редактор, где удобно открывать файлы агента и встроенный терминал — всё в одном окне.
Зачем: чтобы править файлы агента (личность, настройки) и запускать Claude Code.
Node.jsплатформа
Платформа, на которой запускается Claude Code. Ставится «внутрь системы», иконки нет, работает в фоне.
Зачем: без неё Claude Code не установится и не запустится. Ставим первой.
Аналогия: Claude Code — игра, Node.js — приставка, на которой она работает.
npmустановщик
«Установщик» программ, который приходит вместе с Node.js. Команда npm install = «установи мне вот это».
Зачем: через него ставим Claude Code и дополнения.
bunдвижок
Быстрый движок, на котором работает плагин связи агента с Telegram. Ставится одной командой.
Зачем: без bun агент-заготовка не запустится. Ставим его в Уроке 1.
Подписка (Claude Pro / Max)доступ
Платный доступ к модели-мозгу. Pro — $20/мес (попробовать), Max — $100/мес (для работы).
Зачем: без подписки агенту «нечем думать», он не запустится.
VPNдля РФ
Программа, которая открывает доступ к заблокированным сайтам. Для России — нужна, чтобы открыть и оплатить Claude.
Зачем: сайт Claude из РФ напрямую не открывается.
start.sh / запуск агентазапуск
Команда bash start.sh — это ЗАПУСК АГЕНТА: тот же Claude, но с подключённым Telegram. Только после неё бот отвечает. А просто claude — это чат в терминале, к Telegram НЕ подключён.
Зачем: различать «чат claude» (правки файлов) и «живого агента в Telegram». Пока окно с start.sh открыто — агент работает.
channel.envнастройки
Файл с настройками агента в папке secrets/: токен бота, твой Telegram ID, память. Слева от = — имя настройки (не меняем), справа — твоё значение.
Зачем: отсюда агент берёт, к какому боту подключаться и где хранить память. Заполняем по готовому промпту.

Личность, навыки, память

CLAUDE.mdличность
Файл-инструкция в папке агента — его «паспорт характера»: имя, как обращаться к тебе, тон, правила. Агент читает его при каждом запуске.
Зачем: чтобы агент звучал как ты задала, а не как безликий робот. Уже лежит в скачанной заготовке — ты его заполняешь, не пишешь с нуля.
USER.mdо тебе
Файл с информацией о тебе (владельце): имя, чем занимаешься, предпочтения. Лежит рядом с CLAUDE.md.
Зачем: чтобы агент знал, кто ты, и отвечал под тебя, а не шаблонно.
Скилл (skill)умение
Готовое умение агента: «писать посты», «планировать», «делать ревью». Папка с инструкцией, которую агент подхватывает по ключевым словам.
Зачем: добавляешь агенту конкретные навыки как кирпичики.
Память (memory)контекст
Способность агента помнить прошлые разговоры — что было вчера, неделю назад. Включается одной настройкой.
Зачем: чтобы не начинать каждый раз с чистого листа.
MCPподключение
Стандарт, по которому агент подключается к внешним сервисам (календарь, почта, базы знаний). «Розетка» для подключения умений.
Зачем: через MCP агент получает руки — может реально что-то делать в других сервисах.
Второй мозг (Second Brain)общая память
ОБЩАЯ память для всех твоих агентов — одна папка second-brain, куда все пишут и откуда все читают. Не личная память одного агента, а общая база команды.
Зачем: что узнал один агент — знают все. Это превращает отдельных ботов в команду с единым контекстом (Урок 4).
Аналогия: общая база знаний компании, куда пишут все сотрудники.
AGENT_LABELметка
Имя-метка агента в общей памяти (например «помощник», «маркетолог»). У каждого агента своё, чтобы видно было, кто что записал.
Зачем: в общем мозге различать записи разных агентов. Путь к памяти у всех общий, а метка у каждого своя.
MCP-серверподключение
Сервис, к которому агент подключается по стандарту MCP, чтобы получить инструменты или знания (база уроков, календарь, почта). «Розетка», в которую агент включается.
Зачем: через MCP-сервер агент получает руки и доступ к данным — может реально работать в других сервисах.
API-ключключ доступа
Секретная строка-ключ от внешнего сервиса (например Groq для голоса, OpenRouter, платформа интенсива). Получаешь на сайте сервиса, вставляешь в настройки агента.
Зачем: по нему сервис узнаёт, что это ты, и пускает агента. Как пароль — никому не показывай.

Хранение и каналы

gitистория
Система, которая хранит историю изменений твоих файлов на компьютере (как «версии» документа).
Зачем: можно вернуться к любой прошлой версии, ничего не теряется.
GitHubоблако
Бесплатный сайт, куда выкладывают папку с проектом (репозиторий), чтобы она хранилась в облаке.
Зачем: это «дом» агента в облаке — оттуда его скачивают и туда сохраняют.
Репозиторийпапка
Папка проекта с историей изменений. Лежит на GitHub. Заготовку агента ты клонируешь (скачиваешь) именно из репозитория.
Зачем: это и есть «дом» агента — внутри плагин, личность, настройки.
clone (клонировать)скачать
Команда git clone — «скачай мне эту папку-репозиторий с GitHub к себе на компьютер».
Зачем: так ты получаешь готовую заготовку агента одной командой.
BotFatherTelegram
Официальный бот Telegram, который создаёт ботов. Пишешь ему /newbot — он делает бота и даёт токен.
Зачем: через него рождается твой Telegram-бот, лицо агента.
Токен ботаключ
Длинная секретная строка, которую даёт @BotFather при создании бота. Ключ от твоего Telegram-бота.
Зачем: связывает агента с твоим ботом. Никому не показывай — это как пароль.
Канал (Telegram)связь
Место, через которое агент принимает и шлёт сообщения. У нас базовый канал — Telegram-бот.
Зачем: чтобы ты общалась с агентом обычными сообщениями в телефоне.
Groqголос
Сервис, который превращает твои голосовые сообщения в текст для агента. Подключается API-ключом (бесплатный на console.groq.com).
Зачем: чтобы агент понимал голосовые, а не только текст (Урок 3, бонус).

Автономия и сервер

Разрешения (permissions) / Bypassавтономия
Claude Code по умолчанию спрашивает разрешение на каждое действие. Режимы: спрашивать всё / молча править / всё молча (bypass). Опасное (удаление, отправка денег) держат под запросом всегда.
Зачем: чтобы агент работал сам, не дёргая тебя на каждую мелочь — но с защитой от опасного.
VPS / сервер24/7
Арендованный компьютер в интернете (~$5/мес), который никогда не выключается.
Зачем: чтобы агент работал круглосуточно, не завися от твоего ноутбука. На старте не нужен — хватит своего компьютера.
SSHвход на сервер
Способ зайти на удалённый сервер из своего терминала командой ssh root@IP-адрес — как «удалённый рабочий стол», только текстом.
Зачем: чтобы управлять арендованным сервером и поставить на него агента (Урок 5). На Mac уже встроен.
Ubuntuсистема сервера
Операционная система для серверов (как Windows или macOS, но для сервера). При аренде VPS выбираешь её.
Зачем: на ней работает арендованный сервер с твоим агентом.
tmuxфоновый запуск
Программа, которая держит агента запущенным на сервере, даже когда ты закрыла окно или отключилась.
Зачем: чтобы агент на сервере работал 24/7 без открытого терминала (Урок 5).
Конфликт 409ошибка
Ошибка Telegram, когда один и тот же бот запущен ДВАЖДЫ (например на ноуте и на сервере одновременно). Telegram пускает только одного — и бот начинает молчать.
Зачем знать: правило «один бот = один запуск». Перед запуском на сервере останови агента на ноуте.
