Инструкция для маркетинг-агента

Залетающий контент в Threads → автопостинг через агента

Как агент разбирает залетевшие ветки конкурентов, генерит посты под пересылки в твоём стиле и сам публикует их в Threads через официальный API. Настраиваешь один раз — дальше присылаешь тему и получаешь готовые посты (или автопостинг по расписанию).

Что на выходе Пачка постов для Threads, заточенных под пересылки и охват (хук → ценность → мысль, которую хочется репостнуть), в твоём Tone of Voice — и агент, который сам публикует их через официальный API Meta, разово или по расписанию.

Главное про Threads: что реально репостят

На Threads охват растёт не с рекламы, а с постов, которые вызывают реакцию «согласен / зацепило / хочу сохранить». Работают: личные инсайты и истории с выводом, спорные тезисы (мнение против мейнстрима), полезные списки, короткие сильные наблюдения. Не работает: сухая реклама, длинные простыни без хука, «купите у меня». Пайплайн генерит именно репостный формат.

Часть 2. Что даёшь агенту каждый раз

  • Тему или инфоповод — о чём пачка постов (например «ошибки новичков в AI-агентах», «мой кейс», «разбор тренда»).
  • Сколько постов сгенерить (например 5-7 вариантов).
  • (один раз) Tone of Voice и примеры залетевших веток, если ещё не давала.

Часть 3. Пайплайн по шагам (что делает агент)

  1. Разбирает залетевшие ветки
    По твоим примерам выделяет рабочие паттерны: тип хука (вопрос / спорный тезис / цифра / история), длина, формат, темы, что вызывает пересылки.
  2. Собирает «формулу» под твою нишу
    Фиксирует 3-5 рабочих форматов хуков и структур — как шаблоны, по которым строит посты.
  3. Генерит пачку постов
    По теме и формулам пишет 5-7 вариантов постов, каждый под пересылку: сильный первый экран (хук) → ценность/мысль → строчка, которую хочется репостнуть. В твоём Tone of Voice.
  4. Показывает тебе на утверждение
    Отдаёт варианты списком, ты выбираешь/правишь. Ничего не публикует без твоего «ок» (пока не включишь автопостинг).
  5. Публикует через официальный API
    Одобренные посты выкладывает в Threads по токену — сразу или ставит в расписание (например 1-2 поста в день).
  6. Смотрит, что зашло
    По охвату/пересылкам отмечает удачные форматы и в следующей пачке усиливает то, что сработало.
Почему это работает Ты не гадаешь, что запостить. Агент опирается на то, что УЖЕ репостят в нише, и переупаковывает под твой голос. Регулярность + рабочие форматы = рост охвата.

Часть 4. Готовые промпты для агента

Промпт 1 — разовая настройка (формула + подключение)Ты мой маркетинг-агент для Threads. Настрой пайплайн. РАЗОВАЯ НАСТРОЙКА: 1. Подключи официальный Threads API (Meta for Developers). Мой access token — [где я его положила]. Токен держи в защищённом файле, в чат не выводи. Проверь, что можешь опубликовать тестовый пост. 2. Запомни мой Tone of Voice: [тон / обращение / запретные слова / 3-5 примеров моих фраз]. 3. Вот примеры залетевших веток в моей нише: [ссылки/тексты]. Разбери их: какой хук, длина, формат, тема, что вызывает пересылки. Собери 3-5 рабочих формул-шаблонов и сохрани в память. Дальше жди от меня тему для пачки постов.
Промпт 2 — сгенерить пачку постовСгенери [5-7] постов для Threads на тему: [тема/инфоповод]. Используй мои рабочие формулы и Tone of Voice. Каждый пост — под пересылку: сильный хук в первой строке, ценность/мысль, финальная строчка, которую хочется репостнуть. Формат Threads: коротко, живо, без рекламы «в лоб». Выдай варианты списком — я выберу и поправлю. Пока не публикуй.
Промпт 3 — опубликовать / поставить в расписаниеОпубликуй эти посты в Threads через официальный API: [номера/тексты]. Либо поставь в расписание: [например по 1-2 поста в день в 12:00 и 19:00]. После публикации пришли ссылки на посты. Через [день/неделю] собери по ним охват и пересылки и скажи, какие форматы сработали лучше.

Часть 5. Технические детали (для тех-агента)

  • Официальный API: Threads API через Meta Graph — создать App на developers.facebook.com, добавить Threads API, пройти OAuth аккаунтом Threads, получить long-lived access token.
  • Публикация: двухшаговая — создать media container (текст/картинка/карусель), затем publish. Есть эндпоинты для текста, изображений, каруселей.
  • Пример-репо (официальное, Meta): github.com/fbsamples/threads_api — рабочий пример публикации.
  • Расписание: агент на сервере 24/7 постит по крону/хукам — автопостинг без твоего участия.
  • Аналитика: Threads API отдаёт метрики (views, likes, reposts) по своим постам — по ним отбираем рабочие форматы.
Не использовать junhoyeo/threads-api, Danie1/threads-api и другие reverse-engineered библиотеки со входом по логину/паролю — оба заброшены (архив) и грозят баном аккаунта. Только официальный API.
Пайплайн-инструкция для интенсива «AI-команда на Claude». Токен Threads — личный, хранится защищённо. Только официальный Threads API (Meta). Посты всегда проходят через личный Tone of Voice — без него будет чужой голос.
